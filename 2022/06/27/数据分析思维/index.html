数据处理

  1. 缺失性和合理性检验，剔除无效字段（50%）
  2. 数值型字段相关性验证（0.7以上高相关性），同类型数值相关性都高，则选择一种
  3. 字符串字段离散化处：打分法，独热编码
  4. 数据标准化，考虑量级，量纲

模型评估

数据分析方法

指标体系建立

定义及选取原则

（指标体系建设，宣传以及实施）

  1. 前中后期，指标体系不一样
  2. 分析师牵头业务方协助
  3. 各维度反映业务核心状况，多维度
  4. 大实施框架

特性

  1. 根本性：核心数据理解到位和准确
  2. 可理解性：指标配上业务解释性，定义等
  3. 结构性：业务充分解读，分细指标

构成

派生指标 = 时间段 + 修饰词 +原子性指标

步骤

  1. 理清业务阶段和方向
  2. 确定核心指标（正确性）
  3. 指标核心维度拆解：对核心指标进行公式计算，按照业务路径或业务模块进行拆解
  4. 指标宣贯，存档，落地：
    • 当面触达所有相关业务接口人
    • 对指标口径和业务逻辑进行详细描述存档
    • 建核心指标相关报表

流量分析

看数据的过程，指广义的流量，即所有的流量

如：用户从哪来，经过什么过程，产生什么价值，为何流量波动了

即：渠道分析，转化分析，价值分析，波动分析

渠道分析

渠道来源

渠道衡量

关键指标

前期看有效用户数和次留，中期看次留、7日、30日，后期看ROI

避免刷量嫌疑，需要看有主动行为的用户数

分析方法
  • 结构分析：先按一级渠道拆解，再按二级渠道拆解
  • 趋势分析：看每个渠道变化趋势，量级和留存
  • 对比分析：不同渠道间趋势对比
  • 作弊分析：用户行为和机器学习

转化和价值分析

漏斗分析法
  • 基础属性：手机品牌，地域，IMEI等用户画像
  • 行为属性：入口，时段，用户活跃度，用户标签
功能模块分析
  • 功能渗透率：功能用户数/大盘用户数
  • 功能留存率：第一天使用功能第二天也使用功能用户数/第一天使用功能用户数
  • 功能大盘留存率：第一天使用功能第二天也使用功能用户数/第一天大盘用户数
价值分析
  • 功能核心用户数：符合某种要求的功能用户数
    • （使用次数，使用时长，使用天数，某种行为）
  • 功能对大盘贡献：如，对大盘留存提升的贡献
    • 功能A对大盘留存的提升贡献=功能A渗透率 * 功能A大盘留存提升数
  • 功能带来收入对比

波动分析

日活波动
  • 外部：行业变化，竞品变化
  • 内部：数据统计问题，用户基础属性，用户行为属性
留存
  • 新用户：与渠道和渠道过程有关
  • 老用户：核心功能用户留存，非核心非功能用户留存

路径分析

（排查异常，提出问题）

在用户所有操作行为中发现产品设计初可能不知道、但非常有意思的用户前后行为—-路径分析

（基于数据本身发现的，产品可能不清楚，但是复合用户习惯的路径）（先有数据再验证假设）

与漏斗分析的区别：漏斗分析认为设定一条或若干条漏斗（先有假设再数据验证）

日志：路径分析基于底层日志，记录了用户最详细的行为信息

筛选，关联，标准化及画图，启发

路径分析都是以功能点的时序整体分析为主，只有指标没有维度，精细化运营必须进行维度拆分

竞品分析

  • 产品准备进入行业，先行业内竞品分析清楚，侧重行业规模和前景的分析

  • 产品处于下降阶段，看竞争对手在做什么，侧重头部玩家的玩法分析

  • 产品处于瓶颈阶段，看竞争对手的数据和功能迭代，持续监控对手数据，找突破

  • 产品处于快速上升，一般不做竞品分析

  • 确定分析的目的

    • 尝试进入新行业，评估可行性：偏行业趋势，市场规模，财务收入
    • 看竞品功能、玩法和数据，学习优点：功能体验，运营手法，具体数据，落地性强
    • 看竞品迭代的功能、玩法和数据，揣摩竞品做法，预防：版本迭代，战略重心，管理层需要

  • 挑选竞品，对比分析：挑选，进行对比运营手法分析，看宏观微观数据

  • 给初步分析结论：

营销活动分析

  • 分析连贯性：前中后期都要分析
  • 分析对比性：活动与活动之间对比
  • 分析公正性：评判有一套商定好的标准

前期

与运营方多沟通，了解活动整体情况：5w2h

  • 商定活动目标
  • 沟通数据埋点
  • 提前搭建指标体系和报表
    • 找到关键指标，按照基础属性和行为属性拆解
  • 定好输出格式

中期

观察数据

  • 第一天数据很关键
    • 指标体系报表数据是否异常
  • 一到三天数据
    • 预估活动目标的完成度，是否需要调整
  • 定期输出活动战报，让所有人了解情况
  • 一周后数据复盘
    • 同步管理层

后期

报表建设，公正性

  • 对大盘影响，同比，环比
  • 短期效果
  • 长期效果
  • 存在问题及优化

了解活动本质和意义，从这方面入手阐述贡献

用户增长

  • （拉新角度）AARRR —-获取-激活-留存-变现-推荐
  • （留存角度）留存-变现-推荐-拉新-激活
  • （变现角度）变现-推荐-拉新-激活-留存

难落地思维

  • 魔法数字（用户主动行为）
  • 优化渠道结构
  • 流失召回

增长思维

  • 北极星指标：找到最核心的指标
    • 找对指标后进行拆解
    • 拆解后指标与每个团队进行KPI挂钩
  • AB测试：公正性和快速反馈性
    • 原则：基于数据分析，要看结果数据和过程数据，排坑第一步




通过漏斗模型发现引导问题
身份证和押金前后顺序是产品设计层面，本质是AB测试
转化率漏斗很长，要做各种维度拆解


锻炼方法

  • 独立思考和辩证性思维
  • 多研究用户数据
  • 学习他人优点
  • 所有分析增长都要靠产品、运营闭环
  • 成体系才是可传播、可继承

问题的定义

先分享：分享给别人，别人有不一样的理解、疑问、质疑和新想法，增强自身的认知程度
再使用：结合自身实际情况进行使用

提了很多需求，问题是否找到
问题拆到第几层，拆解深度是否足够
问题拆解到最后一层，执行是否有问题

问题识别

二八定理，找出主要问题

  • 列出所有问题，头脑风暴
  • 最大概率法则测试（是否解决一个问题，其它问题迎刃而解
  • 遇到困恼请教别人

问题拆解

  1. 按照自己理解进行拆解，想到多少是多少
  2. 就自己的拆解与业务方熟悉业务的人进行沟通
  3. 修改第一步，请教leader
  4. 再改一次，汇报业务方leader

SQL数据获取

前期

  • 了解业务方和研发说的哪份表和日志
  • 了解表和日志筛选条件，为什么
  • 了解表和日志有什么问题，例：是否缺失某天数据
  • 验证现在是否存在问题，测试跑数据

报告撰写

（多看优秀报告，获得别人反馈

原则

  • 主题一脉相承
  • 通俗简单易懂
  • 结论闭环先行
    • 基于什么数据
    • 发现什么结论
    • 基于这个结论的建议
    • 基于这个建议的产品落地项

组成部分

  • 背景：为何做，即问题识别
  • 分析结论：面向管理层
  • 分析狂阶：问题拆解
  • 第一个关键点结论
  • 第一个关键支撑数据
  • 。。。
  • 整体结论：再汇总一次结论
  • 落地项：产品落地，具体到时间人预期效果

AB测试

流程

  • 根据数据分析得到某建议项
  • 根据建议项对落地进行AB测试
  • 根据落地项，研发开发设计
  • 数据采集
  • 跟进AB测试效果，显著性95%以上

灰度按照5%，10%，20%，50%，100%控制

注意事项

  • AB两组是否真相同（尽可能一个变量不同
  • 验证策略是否生效
  • AB测试评估指标体系（综合性指标体系
  • 多观察几天数据

AB测试存档规划（5W2H，预期和实际效果
